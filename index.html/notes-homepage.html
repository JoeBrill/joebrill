Adding images

Important: all images should have the following attributes; image, alt, width and height.

        <img src="image.jpg" 
             alt="woman typing on computer" 
             width="600" 
             height="300">

In some instances, such as when adding a companies logo, the alt attribute should be left blank for the user experience.

File Formats

GIF

  • Good for compression large areas of a single color
  • Limited color space of 256 colors
  • Can do transparency, with jagged edges
  • Can have multiple frames, and make a little movie

SVG

  • Logos, icons, etc
  • Vector file
  • Prefered over all other types

JPG

  • Image format for compressing photos
  • Can be compressed by simplifying colors used
  • Make sure to resize and compress JPG

PNG

  • Images that need transparency
  • Good at compression
  • Photos & images
  • Sometimes better at compressing images than the JPG

Image Responsiveness

To account for different screen sizes, browser speeds, and machine capabilities, you can use HTML to send different sized images to the browser.

One of the more simple ways to do this can be seen bellow:

Navigation Bars

Below is an example of an unordered navigation bar setup. Note that it appears as a list without styling from CSS. It is very important when making a navigation bar to mark it up using ARIA roles so that the browser knows what it is.

  • Home
  • People
  • Prices
  • Contract
        <nav>
            <ul class="navbar" aria-label="main-menu">
                <li><a href="/">Home</a></li>
                <li><a href="/People">People</a></li>
                <li><a href="/prices">Prices</a></li>
                <li><a href="/contract">Contract</a></li>
            </ul>
        </nav>

Following is an example of an ordered list navigation (shown using a breadcrumb trail):

  1. Home
  2. Blog
  3. March
  4. March 9th updates
        <nav aria-label="Breadcrumb">
            <ol class="breadcrumbs">
                <li><a href="/">Home</a></li>
                <li><a href="/people">Blog</a></li>
                <li><a href="/contact">March</a></li>
                <li>March 9th updates</li>
            </ol>
        </nav>

At the bottom of wepages you can often find footers. As these are not key to the material of the page being viewed it is marked as unimportant using ARIA-labels.

Dictionary

Attributes (HTML)

  • Global Attribute
    • Applies to any HTML element
    Order of attributes never matters

  • Class Attribute
    • Allows us to target all elements with that class in our CSS or JavaScript
    <p class="intro">
    This is the introduction.
    </p>

  • id Attribute
    • Allows us to target a unique element with that id in our CSS of JavaScript
    <p id="article-intro">
    This is the introduction.
    </p>

    because id is so specific developers tend to stick to classes as the
    specificity can be a challenge to manage

    Are particularly helpful for addressing elements with JavaScript or a targeted link

  • Content Editable
    • sets specific elements to be editable by users.
    Click here to edit
  • lang
    • Sets the language (more detail later)

  • dir
    • Sets the direction of text.
